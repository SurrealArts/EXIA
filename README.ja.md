# EXIA: Discord モデレーションボット

EXIA は、**Node.js 26** と **Discord.js v14** で構築されたモジュール式自動モデレーションボットです。設定可能な脅威検出モジュールに基づいて、重み付けされた圧力システムを使用して制裁（警告 → ミュート → キック → BAN）をエスカレーションし、永続的な SQLite データベースに支えられています。このプロジェクトは、既存のアンチスパムおよびモデレーション手法を継承し、インスピレーションを得ています。[User319183/Discord-Anti-Spam-Bot](https://github.com/User319183/Discord-Anti-Spam-Bot)、[Michael-J-Scofield/discord-anti-spam](https://github.com/Michael-J-Scofield/discord-anti-spam)、[Elaina69/Auto-Ban](https://github.com/Elaina69/Auto-Ban) に感謝します。

## 特徴

- **i18n / 多言語対応:** 内蔵ロケールシステムによるサーバーごとの言語選択（`/language`）。日本語と英語に対応
- **レイド保護:** 制裁スパイクしきい値に基づく自動エスカレーション（3段階）。`/raid` による手動オーバーライド可能
- **圧力システム:** 60秒の減衰サイクルを持つユーザー/ギルドごとのインメモリ圧力スコア。クリティカルモジュールトリガー時のファストトラック（即時BAN）
- **構成プロファイル:** ギルド設定全体のスナップショット、復元、エクスポート、インポート。初回使用時に Standard プロファイルが自動適用
- **テレメリーキュー:** 設定可能なチャンネルへのバッチ埋め込みログ。アクション/フラグ/コマンドカテゴリでグループ化
- **DMでの制裁通知:** 警告、ミュート、キック、BANの前に、ユーザーに理由を説明するダイレクトメッセージを送信

## 脅威検出モジュール

各モジュールは、その設定に基づいてシステムに圧力を加えます。モジュールが追加されるにつれて、ここにも追加されます。

### Velocity Bucket

トークンバケットレートリミッター。ユーザーごとのメッセージ頻度を追跡し、高速マルチチャンネルスパムを検出します。メッセージレートがバケット容量を超えると圧力を適用します。

### Regex Sandbox

Worker スレッド内でカスタム正規表現パターンに対してメッセージ内容を評価します（ReDoS セーフ）。招待URL、ブラックリスト用語、その他のパターンベースのコンテンツなどの設定可能なルールに一致します。

### Honeypot Trap

制限されたチャンネルを指定します。管理者以外がそこにメッセージを送信すると、圧力システムのファストトラックメカニズムを介して即時BANがトリガーされます。

### User Profile

ユーザーのアカウントの経過期間とアバターステータスを監査します。アカウントが若い、アバターがない、またはその両方の場合に、他のモジュールからの圧力を増幅する乗数（1.0x～2.0x）を生成します。

### Mention Guard

@everyone、@here、ロールメンションをモデレーター以外のユーザーから検出します。繰り返しの違反でエスカレーションし、クリーンなメッセージを送信すると減衰する動的乗数（3.0x～5.0x）を適用します。メッセージ管理権限を持つユーザーは除外されます。

## デプロイ

### 前提条件

- **Node.js 26**（ローカル開発用）または **Docker**（コンテナ化デプロイ用）
- 以下のインテントが有効な [Discord Developer Portal](https://discord.com/developers/applications) からの **Discord ボットトークン** と **クライアントID**:
  - `Presence Intent`、`Server Members Intent`、`Message Content Intent`
- **ボット権限:** [Discord Permission Calculator](https://discordapi.com/permissions.html) を使用して以下のデフォルト権限を設定するか、権限整数 `274878000150` でボットを招待:
  - `View Channels`、`Manage Channels`、`KickMembers`、`BanMembers`、
  - `Send Messages`、`Send Messages in Threads`、`EmbedLinks`、`Manage Messages`、`Read Message History`

### 1. 環境設定

`.env.example` から `.env` ファイルを作成:

```env
CLIENT_ID=your_client_id
TOKEN=your_bot_token
VERSION=1.2.1

LOG_WITH_TIME=true
LOG_TIMEZONE=UTC
```

### 2. Docker デプロイ

```bash
docker compose up -d --build
```

ボットは起動時に6つのスラッシュコマンドをグローバルに登録:

- `/config` — モジュール、しきい値、正規表現ルール、ハニーポット、ログチャンネルを管理、設定を表示
- `/actions` — 異議申し立て/再参加リンク、手動BAN、メンバースキャンの更新
- `/language` — サーバーの優先言語を設定
- `/profiles` — 構成プロファイルの作成、一覧表示、適用、エクスポート、インポート、削除（Standardはロック済み）
- `/raid` — レイドステージ（0–3）の設定、現在のステータスを確認
- `/debug` — システム状態の完全表示: モジュール、しきい値、アクティブな圧力スコア、テレメリーキュー

SQLite データは名前付き Docker ボリューム (`exia_data`) に永続化されます。

### 3. ローカル開発

```bash
pnpm install
pnpm run dev
```

`SIGINT`/`SIGTERM` でのグレースフルシャットダウン（クライアントの適切な破棄を含む）。

## データ

すべてのデータは `data/exia.db`（SQLite、WAL モード）に保存されます。WAL ファイル（`.db-wal`）には最新の書き込みが含まれています。標準の SQLite ビューアで現在の状態を表示するには、WAL をマージする必要があります。

# 個人的なメモ

それぞれに特定のケースをキャッチする際に問題がある複数の異なる Discord ボットを使用することについて、しばらく前から懸念していました。私はこれらの方法を1つのボットに「統一」しようとしてこのプロジェクトを作成しましたが、スケールアップを考えるほど、私には寄せ集めのように思えてきます。だからこそ、これらの方法を緩やかに「モジュール」として分類しています。しかし、もう1つのインスピレーションを考慮すると、私は「重み付き通知システム」を作成しました。これは一連のプロトコルと重み付き意思決定です。このプロジェクトはそこからアーキテクチャのインスピレーションを得ています。将来的には、私が考案したいくつかの方法のアイデアに貢献したリポジトリを記載する専用セクションを設ける予定です。

そうそう、それと...私はこのアーキテクチャを、しばらくやってきたオープンソースのDiscordボットテンプレートとは異なるものにしました。過去のプロジェクトを更新することで知られていないことは自覚していますが、これは特に、コードが実行されるときに追加した極端な冗長性のおかげで、簡単に戻ってこれるようにするためのスタートです。
